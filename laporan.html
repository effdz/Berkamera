BAB 1. PENDAHULUAN

Proyek ini merupakan implementasi website e-commerce yang berfokus pada penjualan kamera dan aksesoris fotografi. Website "Berkamera" dirancang untuk memenuhi kebutuhan fotografer profesional maupun pemula dalam mencari peralatan fotografi berkualitas tinggi. Platform ini menggabungkan fitur e-commerce modern dengan konten edukatif berupa artikel dan tutorial fotografi.

1.1. Deskripsi Topik

Berkamera adalah platform e-commerce yang mengkhususkan diri dalam penjualan kamera, lensa, dan aksesoris fotografi. Proyek ini mencakup pengembangan website lengkap dengan fitur-fitur berikut:

Scope Project:

  • Frontend User Interface: Desain responsif untuk desktop dan mobile
  • E-commerce Functionality: Katalog produk, keranjang belanja, sistem checkout
  • Content Management: Sistem artikel dan blog fotografi
  • User Management: Registrasi, login, dan profil pengguna
  • Admin Panel: Dashboard admin untuk mengelola produk, pesanan, dan konten
  • Payment Integration: Simulasi sistem pembayaran multi-metode
  • Order Management: Tracking pesanan dan riwayat transaksi

Teknologi yang Digunakan:

  • HTML5, CSS3, JavaScript (Vanilla)
  • Local Storage untuk data persistence
  • Responsive Design dengan CSS Grid dan Flexbox
  • Font Inter dari Google Fonts
  • SVG Icons untuk interface elements

1.2. User Story

Sebagai Customer:

  • Saya ingin dapat melihat katalog produk kamera dan aksesoris dengan informasi detail
  • Saya ingin dapat menambahkan produk ke keranjang belanja dan wishlist
  • Saya ingin dapat melakukan checkout dengan berbagai metode pembayaran
  • Saya ingin dapat membaca artikel dan tutorial fotografi
  • Saya ingin dapat melacak status pesanan saya
  • Saya ingin dapat mengelola profil dan riwayat pembelian

Sebagai Admin:

  • Saya ingin dapat mengelola katalog produk (tambah, edit, hapus)
  • Saya ingin dapat mengelola pesanan dan status pengiriman
  • Saya ingin dapat mengelola konten artikel dan blog
  • Saya ingin dapat melihat statistik penjualan dan dashboard analytics
  • Saya ingin dapat mengelola data pengguna

1.3. Use Case

Primary Use Cases:

  1. Browse Products - User dapat menjelajahi katalog produk
  2. Product Search & Filter - User dapat mencari dan memfilter produk
  3. Add to Cart - User dapat menambahkan produk ke keranjang
  4. Checkout Process - User dapat melakukan pembelian
  5. User Authentication - User dapat login/register
  6. Read Articles - User dapat membaca konten edukatif
  7. Admin Management - Admin dapat mengelola sistem

Secondary Use Cases:

  1. Wishlist Management - User dapat menyimpan produk favorit
  2. Order Tracking - User dapat melacak pesanan
  3. Product Comparison - User dapat membandingkan produk
  4. Newsletter Subscription - User dapat berlangganan newsletter

1.4. Use Case Specification

Use Case: Checkout Process

  • Actor: Registered Customer
  • Precondition: User sudah login dan memiliki item di keranjang
  • Main Flow:
    1. User mengakses halaman keranjang
    2. User review item dan quantity
    3. User klik "Lanjut ke Pembayaran"
    4. System redirect ke halaman checkout
    5. User mengisi informasi pengiriman
    6. User memilih metode pengiriman
    7. User memilih metode pembayaran
    8. User konfirmasi pesanan
    9. System proses pembayaran
    10. System generate order confirmation
  • Postcondition: Pesanan berhasil dibuat dan user menerima konfirmasi

Use Case: Admin Product Management

  • Actor: Administrator
  • Precondition: Admin sudah login dengan role admin
  • Main Flow:
    1. Admin mengakses dashboard admin
    2. Admin pilih menu "Manajemen Produk"
    3. Admin dapat melihat daftar produk
    4. Admin dapat tambah produk baru
    5. Admin dapat edit informasi produk
    6. Admin dapat hapus produk
    7. System update database produk
  • Postcondition: Data produk berhasil diperbarui

BAB 2. DESAIN SISTEM

Sistem Berkamera dirancang dengan arsitektur frontend yang modular dan responsif. Implementasi menggunakan pendekatan component-based dengan JavaScript modules untuk mengelola fungsionalitas yang berbeda seperti cart management, user authentication, dan payment processing.

2.1. Wireframe

Wireframe sistem dirancang dengan pendekatan mobile-first design, memastikan pengalaman pengguna yang optimal di semua perangkat. Struktur layout menggunakan CSS Grid dan Flexbox untuk menciptakan interface yang responsif dan modern.

Key Design Principles:

  • Consistency: Penggunaan design system yang konsisten
  • Accessibility: Implementasi ARIA labels dan semantic HTML
  • Performance: Optimasi loading dan rendering
  • User Experience: Intuitive navigation dan clear call-to-actions
